# Why I link notes aggressively in Obsidian

The main reason I bother with `[[wiki links]]` instead of folders is that a
folder forces a note into exactly one place, while a link lets the same idea
participate in many contexts. Over time the graph of links becomes more
useful than any folder structure I've tried to maintain by hand.

Two concrete rules I try to follow:

1. A link should point at a concept that could reasonably deserve its own
   note, even if that note does not exist yet. A red link is a todo.
2. I avoid linking every proper noun. If linking "Paris" doesn't help me find
   anything later, it's just visual noise.

The payoff is slow — the first few weeks of linking feel like extra work for
nothing. After about a month, the second-order effect kicks in: opening any
note pulls in a surprising amount of related context from the backlinks pane.
